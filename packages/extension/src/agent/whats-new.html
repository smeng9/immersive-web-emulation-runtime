Immersive Web Emulator 2.0

A more explicit workflow for WebXR emulation.

Pin IWE to your toolbar

The toolbar button is the control center for IWE 2.0: enable or disable WebXR emulation for the current site, open page-level controls, and connect AI agents. If you do not see it, open Chrome's Extensions menu and pin Immersive Web Emulator.

  1. Pin IWE from Chrome's Extensions menu.
  2. Enable the site by clicking the pinned toolbar button.
  3. Use the menu to stop emulation, open controls, or connect AI agents.

Activate emulation per site

IWE 2.0 is intentionally opt-in for the page you are testing. Instead of treating WebXR emulation as always on everywhere, use the pinned toolbar button to activate the current site, closer to explicit test-session tools like SpectreJS.

This keeps normal browsing untouched and makes it clear which page is currently running with the emulator.

What goes where?

Toolbar button: turn emulation on or off for the current site and connect AI agents.

Page overlay: live headset, controller, hand, and button-mapping controls.

DevTools WebXR panel: persistent defaults such as device, environment, input mode, stereo rendering, IPD, and FOV-Y.

Settings reset in 2.0. Set your defaults once in the panel and use the overlay for live page control.

Drive it with an AI agent

Open the pinned toolbar button for agent setup. Point a coding agent at the local @iwer/extension-bridge MCP server and it can drive an emulated WebXR tab after you approve the on-page Allow prompt.
